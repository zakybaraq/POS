# Technology Stack

**Analysis Date:** 2026-04-11

## Languages

**Primary:**
- TypeScript 5.x - Core application language with strict type checking enabled
- HTML5 - Server-rendered templates in `src/pages/` for dynamic page generation
- CSS - Styling assets in `src/public/styles/` served via Elysia static routes

**Secondary:**
- SQL - MySQL database queries via Drizzle ORM

## Runtime

**Environment:**
- Bun (latest) - JavaScript/TypeScript runtime optimized for speed and compatibility with Node.js APIs

**Package Manager:**
- Bun - Primary package manager and task runner
- Lockfile: `bun.lock` (present - generated by Bun)

## Frameworks

**Core Web Framework:**
- Elysia 1.4.28 - Lightweight, type-safe TypeScript web framework
  - Location: `src/index.ts` (main entry point)
  - Purpose: HTTP API routing, server-side rendering, middleware management
  - Features: Request routing, cookie middleware, static file serving
  - Middleware: @elysiajs/cookie 0.8.0 for session token management

**Database:**
- Drizzle ORM 0.45.2 - Type-safe SQL query builder for MySQL
  - Schema: `src/db/schema.ts` (429 lines, fully typed relations)
  - Connection: `src/db/index.ts` with connection pooling
  - Config: `drizzle.config.ts` for migrations
  - Drizzle Kit 0.31.10 - CLI for schema generation and migration management

**Authentication & Security:**
- jsonwebtoken 9.0.3 - JWT token generation and verification
  - Implementation: `src/utils/auth.ts`
  - Token storage: `pos_session` HTTP cookie
  - Default secret: 'pos-secret-key-change-in-production' (dev only)
- bcryptjs 3.0.3 - Password hashing and comparison
  - Usage: User password storage in `users.password` column

**Data Validation:**
- Zod 4.3.6 - TypeScript-first schema validation and parsing

**Dev Dependencies:**
- @types/bun (latest) - TypeScript types for Bun runtime
- @types/jsonwebtoken 9.0.10 - JWT library types
- @types/bcryptjs 3.0.0 - Bcrypt library types
- drizzle-kit 0.31.10 - Database schema tooling

## Database Architecture

**Connection Pool:**
- Type: mysql2/promise
- Max connections: 5
- Queue limit: 0 (unlimited)
- Location: `src/db/index.ts` lines 5-13

**Schema Tables (22 tables):**
- Core: users, orders, orderItems, menus, categories, tables
- Inventory: ingredients, recipes, stockMovements
- Sales: customers, loyaltyTransactions, suppliers, purchaseOrders
- Settings: businessSettings, taxSettings, paymentMethods, receiptSettings, operatingHours
- Operations: employees, shifts, attendance, auditLogs

**Indexing Strategy:**
- Foreign key indices on all relationships
- Status indices (orders, items, kitchen, shifts)
- Search indices (names, emails, phones)
- Timestamps indexed for range queries
